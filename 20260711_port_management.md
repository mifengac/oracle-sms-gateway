# Ubuntu Docker 项目端口管理建议

你现在习惯从 `5001` 开始排端口，这个习惯可以保留，但建议按类型分段，不要完全按项目创建时间递增。

## 推荐分段

| 端口段 | 用途 | 例子 |
|---|---|---|
| `5001-5009` | 主要业务 Web 项目 | `yfjcgkzx`、`zhian-map`、`ltxd_zxxd` |
| `5010-5019` | 公共基础服务 | `oracle-sms-gateway`、统一认证、统一文件服务 |
| `5020-5029` | 定时任务 / 自动化任务 / 后台接口 | 自动巡检、同步任务、监测任务 |
| `5030-5039` | AI / 模型 / Agent 服务 | 语义抽取、图像识别、pydantic-agent |
| `5040-5049` | 预览、临时演示、测试服务 | 临时 demo、灰度版本 |
| `5050-5099` | 预留 | 后续扩展 |

## 命名规则

容器名和项目名保持一致：

```bash
docker run --name oracle-sms-gateway ...
```

不要用 `app1`、`test2` 这种名字。以后排查时直接：

```bash
docker ps --format 'table {{.Names}}\t{{.Ports}}\t{{.Status}}'
```

## 使用原则

- 新项目上线前先查 `PORTS.md`，不要临时猜端口。
- 同一项目的正式版端口固定，不要今天 5003、明天 5008。
- 测试服务用 `5040-5049`，测试完就删除容器并释放端口。
- 公共服务不要夹在普通业务项目中间；短信网关这类服务放 `5010-5019`。
